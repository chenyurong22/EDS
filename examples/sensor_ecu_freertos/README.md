# Sensor ECU FreeRTOS Example — Xaloqi EDS

**Live Sensor Integration on FreeRTOS — 7 DIDs · 4 DTCs**

This is the FreeRTOS counterpart to [`examples/sensor_ecu/`](../sensor_ecu/).  
It demonstrates the complete production pattern — real sensor data, automatic DTC
lifecycle, live DID responses — running on FreeRTOS instead of Zephyr.

The reference for FreeRTOS teams who want live sensor data exposed over UDS diagnostics.

---

## What this example shows

Same sensor monitoring architecture as `sensor_ecu`, but wired to the FreeRTOS
platform HAL:

```
sensor_monitor task (FreeRTOS, 100 ms, priority 5)
    │
    ├── reads sensor data (stub callbacks — plug in your HAL)
    ├── checks thresholds
    └── dtc_database_set_status()   ← DTC lifecycle fully automatic

eds_freertos_start()               ← wires CAN RX callback, creates UDS poll task
vTaskStartScheduler()
```

Customer provides: `can_send` callback, optional NVM ops.  
Xaloqi EDS provides: everything else.

## What this example contains

```
sensor_ecu_freertos/
├── diagnostics_config.yaml      Same YAML as sensor_ecu (RTOS-agnostic)
├── src/main.c                   FreeRTOS entry: eds_freertos_start() + sensor task
├── src/sensor_monitor.c         100 ms monitoring task
├── boards/qemu_cortex_m4/       QEMU Cortex-M4 configuration
└── (generated/)                 Regenerated by CI from same YAML as Zephyr version
```

**Platform:** Builds for QEMU ARM Cortex-M4 in CI.  
**Note:** `generated/` is not committed — CI regenerates it. Run codegen locally after activating your license.
